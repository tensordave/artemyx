# Changelog - v0.8.x

## v0.8.1 - 2026-03-30

### Operations Gate for PMTiles

- **PMTiles datasets disabled in Operation Builder**: dataset dropdowns now show PMTiles entries as disabled options with a "(PMTiles)" suffix; the form validation rejects PMTiles inputs as a fallback; CSS styling mutes disabled options for consistent cross-browser appearance
- **PMTiles operation validation in config parser**: `validateOperations` now checks `input`/`inputs` references against PMTiles dataset IDs; referencing a PMTiles dataset in an operation produces a clear validation error at parse time; the `pmtilesDatasetIds` collection was merged into the existing dataset ID loop to avoid duplicate iteration

### Outputs for Local Files

- **Outputs panel config detection fix**: `hasOutputsInConfig()` and the "Run Outputs" handler both relied on `parseConfig()` which validates output sources against config-declared datasets; locally uploaded files exist only in DuckDB, causing validation to reject them silently; both paths now use lightweight YAML parsing with per-entry structural validation, deferring source existence checks to `checkSourcesExist()` which queries DuckDB directly; the results container empty-state message is now refreshed after Outputs Helper injection so the text updates from "No outputs defined in config" to "Click Run Outputs to generate files" without requiring a panel close/reopen
- **Multi-format dataset export**: the per-dataset context menu (layer panel) now offers Export GeoJSON, Export CSV, Export Parquet, and Export PMTiles as direct menu items; new `exportDatasetAs()` function in `export.ts` routes to the existing worker RPCs (`exportAsGeoJSON`, `exportAsCSV`, `exportAsParquet`, `exportAsPMTiles`) with blob download; PMTiles quick export uses default zoom range 0-14

## v0.8.0 - 2026-03-29

### Keyboard Shortcuts

- **Global shortcut handler** (`shortcuts.ts`): centralized keyboard shortcut system registered via a single `document` keydown listener; shortcuts are disabled when a text input, textarea, or contenteditable element is focused; modifier keys (Ctrl, Cmd, Alt) pass through to the browser; Escape works regardless of focus state
- **Panel toggle shortcuts**: L (layers), P (progress), I (load data), U (upload), C (config editor), T (storage), / (search location), O (operation builder), X (outputs), B (basemap), E (legend); each shortcut toggles its panel open or closed, respecting existing mutual exclusion groups
- **Escape closes all panels**: pressing Esc closes every open panel including floating panels (config editor, operation builder, outputs) and sidebar panels (layers, data, upload, storage, basemap, geocoding); works even when a text input is focused
- **Map navigation**: WASD pans the map (100px per keypress, 100ms animation), R zooms in, F zooms out (200ms animation); held keys repeat via browser key repeat for continuous movement
- **Public `togglePanel()` methods** added to LayerToggleControl, DataControl, GeocodingControl, UploadControl, StorageControl, BasemapControl, ProgressControl, LegendControl; OperationBuilderControl and OutputsControl promoted existing private `toggle()` to public `togglePanel()`; each method mirrors its button click handler to preserve mutual exclusion, input focus, and pointer-down listeners
- **Shortcut key tooltips** on all panel toggle buttons: hovering a control button now shows its keyboard shortcut (e.g., "Load Data (I)", "Switch basemap (B)"); new tooltips added to ProgressControl status row and LegendControl header; close buttons on floating panels (config editor, operation builder, outputs) show "Close (Esc)"

### ARIA Labels

- **`aria-label` on icon-only buttons**: all control toggle buttons, config editor action buttons (edit, import, generate, export, run, clear, close), operation builder clear/close, outputs close/config, progress clear/minimize, scale unit/coordinate format toggles, advanced options toggle, and layer row buttons (visibility, menu, move up/down with contextual dataset names); dynamic labels update alongside dynamic title changes (edit mode toggle, clear confirm state)
- **`aria-expanded` on toggle buttons**: all 12 control toggle buttons and advanced options toggle track open/closed panel state; updated on every open, close, and toggle path
- **`role` attributes on interactive elements**: `role="menu"` on context menus, `role="menuitem"` on items, `role="separator"` on dividers; `role="dialog"` with `aria-modal` and `aria-label` on all modal dialogs (error, CRS prompt, file prompt, confirm); `role="button"` with keyboard activation (Enter/Space) on legend header and progress status row; `role="application"` with `aria-label` on the MapLibre canvas
- **Screen reader announcements** for layer state changes: visually-hidden `aria-live` region in the layer control announces visibility toggles, layer reorders, and layer deletions

### Button Text Labels

- **Label toggle control** (`label-toggle-control.ts`): top-left toggle button (N shortcut) that shows or hides text labels beside the 9 main map control buttons; icon swaps between TextT (labels off) and TextTSlash (labels on) for clear visual state feedback
- **Post-hoc label injection** (`ui/button-labels.ts`): DOM utility that reads `aria-label` from each `.control-btn` and appends a `<span class="control-btn-label">` with shortened display text (e.g., "Load Data" becomes "Data", "Toggle layers" becomes "Layers"); runs once after all controls are mounted
- **CSS-driven visibility**: single `show-labels` class on the map container controls label display; buttons expand to `width: auto` with padding and gap when active; labels force-hidden on mobile viewports (max-width: 767px)
- **Responsive panel offsets**: `.control-panel--left` and `--right` changed from hardcoded `40px` to `calc(100% + 11px)`, adapting to button width whether labels are shown or not
- **Preference persistence**: toggle state saved to `localStorage` (`artemyx-show-labels`) and restored on page load

### Focus Management

- **Focus trap utility** (`utils/focus-trap.ts`): reusable module that traps Tab/Shift+Tab within a container element; re-scans focusable elements on every Tab press to handle dynamic content (layer list rebuilds, config editor mode switches); skips trapping inside textareas to preserve Tab-to-indent behavior in the YAML editor; filters hidden elements (e.g., `display:none` file inputs) from the focusable set
- **Focus trapping on complex panels**: config editor, layer panel, operation builder, and outputs panel create and activate a focus trap on open; trap is deactivated on close; `updateElements()` called after DOM mutations (edit mode transitions, style view switches, input rebuilds, body re-renders); `focusFirst()` moves focus into the panel on open so the trap captures Tab events from the start
- **Focus trapping on modal dialogs**: all four dialog types (error, CRS prompt, file prompt, confirm) trap focus within the dialog box; trap deactivated and focus restored to the previously focused element on close
- **Focus restoration on all panels**: every panel stores `document.activeElement` on open and restores it on close; anchored panels (data, upload, geocoding, basemap, storage) clear the stored reference when closing via click-outside so focus follows the user's pointer target instead of snapping back to the trigger button
- **`:focus-visible` styles**: keyboard-only focus ring (`2px solid #3388ff`) on control buttons, dialog buttons, basemap options, layer action buttons, config editor action buttons, outputs buttons, storage action buttons, upload browse button, geocoding result items, operation builder selects and run button

### Feature Popups Fix

- **Single click popup for overlapping layers**: replaced per-layer click handlers with a centralized map-level click handler mirroring the existing hover tooltip pattern; `attachFeatureClickHandlers` now registers layer IDs into a module-level `clickRegistry` (`Set<string>`) and attaches a single `map.on('click')` handler that calls `queryRenderedFeatures` to select the topmost feature across all registered layers; one shared `maplibregl.Popup` instance replaces the per-call-site popup instances that caused multiple popups to open simultaneously
- **Hover popup parameter removed**: `attachFeatureHoverHandlers` return type changed from `maplibregl.Popup` to `void`; the click handler now accesses the shared hover popup directly via module-level state (`sharedHoverPopup`) to dismiss it on click, eliminating the `hoverPopup` parameter that was threaded through 11 call sites
- **Click registry cleanup**: `removeFeatureHandlers` and `clearAllFeatureHandlers` now clear both the hover and click registries, preventing stale layer IDs from causing `queryRenderedFeatures` errors after dataset deletion
- **Cursor handler deduplication**: per-layer `mouseenter`/`mouseleave` cursor handlers are only registered once per layer via the click registry, preventing duplicate registrations on rename or re-registration paths

### PMTiles Output Memory Optimization

- **Eager intermediate data release** (`pmtiles-writer.ts`): FeatureCollection is nulled after geojson-vt consumes it and tile index is nulled after the encoding loop, with yield points between stages to allow GC before the next allocation; reduces peak worker memory by preventing the source data, tile index, and archive buffer from coexisting
- **Incremental tile buffer release** (`pmtiles-writer.ts`): individual compressed tile buffers are nulled after being copied into the final archive during `buildPMTilesArchive()`, avoiding holding both the tile array and assembled archive simultaneously
- **Extraction pipeline cleanup** (`pmtiles-reader.ts`): `extractPMTilesAndRebuild()` releases extracted feature data before passing to the writer so extraction results do not overlap with re-tiling memory
- **Blob constructor fix** (`output-executor.ts`): changed `new Blob([buffer.buffer as ArrayBuffer])` to `new Blob([buffer as BlobPart])` to pass the Uint8Array directly instead of dereferencing the backing ArrayBuffer
- **Worker crash diagnostics** (`worker.ts`, `client.ts`): top-level `error` and `unhandledrejection` listeners in the worker attempt to report OOM errors before the process dies; main-thread `onerror` handler now includes which RPC was in flight when the crash occurred

### Outputs Helper

- **Output builder form** (`outputs-control.ts`): collapsible "Add Output" section inside the existing Outputs panel; format dropdown (GeoJSON, CSV, Parquet, PMTiles) drives dynamic source and params sections; reuses `ob-*` form field classes from the Operation Builder for visual consistency
- **Multi-source rows for PMTiles**: when format is PMTiles, the source field switches to add/remove rows with individual dropdowns and remove buttons; selections are preserved across row additions and removals; single-source mode for all other formats
- **Dynamic PMTiles params**: minzoom/maxzoom number inputs, layer name (hidden in multi-source mode since each source ID becomes a layer), extract zoom (shown when a PMTiles dataset is selected as source), and bounding box grid with a "Use current view" button that fills all four values from the current map viewport
- **Live YAML preview**: collapsible Shiki-highlighted YAML preview (150ms debounced) showing the exact output config that will be injected; follows the Operation Builder's toggle pattern
- **"Add to Config" injection**: parses the current config YAML, appends the new output entry to the `outputs` array, re-serializes, and pushes the updated YAML back to the Config Editor via a new `updateYaml` callback; validates the output structure before injection using the existing `validateOutput()` function; shows success/error status inline
- **Panel height fix**: `ensurePositioned()` no longer freezes panel height, allowing the Outputs panel to auto-expand when the builder is opened in side-by-side mode with the Config Editor; `max-height: 80vh` added to prevent viewport overflow
- **Panel scroll fix**: wrapped builder and body sections in a single scroll container so the entire panel content below the header scrolls when it exceeds the viewport cap; previously the builder section sat outside the scrollable area, causing it to clip without scrolling when many source rows were added; resize handle now allows expanding to 95% viewport height for large multi-source configurations
