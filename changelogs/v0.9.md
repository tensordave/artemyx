# Changelog - v0.9.x

## v0.9.1 - 2026-04-12

### Legend support for deck.gl layers

- **deck.gl legend entries**: `buildDeckLegendGroups` in `legend-control.ts` queries the renderer registry for datasets with deck.gl layers, reads authoritative visibility from the manager via new `getLayerEntry` accessor, and creates solid color legend entries from the dataset's stored color; datasets already covered by MapLibre legend groups are skipped to prevent duplicates
- **Legend refresh on panel update**: `refreshPanel` in `layer-control.ts` now calls `legendControl.refresh()` at the end so the legend rebuilds after config loads, file uploads, and other dataset changes that don't trigger MapLibre's `styledata` event
- **Legend refresh on visibility toggle**: visibility toggle callback in `layer-control.ts` calls `legendControl.refresh()` so hiding/showing deck.gl layers immediately updates the legend

### Dataset preview panel

- **DatasetControl**: new top-level map control using the database Phosphor icon, positioned between OutputsControl and StorageControl in the top-right; keyboard shortcut T (daTasets); opens a draggable/resizable floating panel listing all datasets currently in DuckDB; visible in both in-memory and OPFS modes; mutual exclusivity with other right-side inline panels (Data, Upload, Operation Builder, Storage); follows the established OutputsControl floating panel pattern with drag, resize, focus trap, and mobile bottom-sheet
- **Dataset row display**: each row shows the dataset ID (slug) in monospace as the primary identifier, the human-readable name as a subtitle when it differs from the ID, feature count, and a color border matching the dataset color; gives users visibility into the config-level identifiers that DuckDB and YAML configs reference
- **Table preview**: table icon on non-PMTiles dataset rows opens a separate floating panel (z-index 25, above the dataset panel) showing the first 50 rows of flattened properties in a plain HTML table; monospace font, sticky column headers, row numbers, alternating row backgrounds, horizontal scroll for wide datasets, null styling, and long-value truncation; data queried via new `getPreviewRows` worker RPC that reuses the existing `getAllPropertyKeys` and `buildFlattenedSelect` helpers in `features.ts`
- **Geo layer cross-reference**: map-pin icon on spatial dataset rows expands an inline sub-list below the row showing each MapLibre and deck.gl layer that renders the dataset; each entry displays the human-readable layer name, renderer badge (MapLibre or deck.gl), and layer type (fill, line, circle, symbol); sub-list pushes subsequent rows down and toggles on repeat click

### Dataset button status colour

- **Icon colour by dataset state**: DatasetControl button SVG fill updates to reflect loaded data: blue (#3388ff) when no datasets are loaded, teal (#14b8a6) when only tabular (non-spatial) datasets exist, green (#22c55e) when any spatial dataset is present; follows the same `svg.setAttribute('fill', color)` pattern as StorageControl
- **Update on all data paths**: icon colour refreshes after OPFS session restore, config pipeline completion, URL load via DataControl, file upload via UploadControl, and dataset deletion via LayerControl context menu; DataControl and UploadControl receive an `onDatasetChange` callback; LayerToggleControl receives a `datasetControl` reference via `setDatasetControl` setter

### Non-spatial dataset ingestion

- **Non-spatial loader fallback**: CSV, Parquet, and JSON array loaders no longer throw when no geometry column is detected; instead they return features with null geometry and a `nonSpatial` flag, allowing any tabular file to be loaded into DuckDB as a table-only dataset
- **Non-spatial database path**: `loadGeoJSON` in `datasets.ts` accepts a `nonSpatial` option that inserts rows with NULL geometry and marks the dataset with `is_spatial = false` in the datasets table; new schema migration adds `is_spatial BOOLEAN DEFAULT true` column
- **Worker pipeline routing**: `workerLoadFromResponse`, `workerLoadFromBuffer`, and the non-paginated JSON path in `workerLoadFromUrl` check for `nonSpatial` loader results or explicit `tableOnly` options and route to `workerLoadNonSpatial`, which skips GeoJSON serialization, bounds calculation, and CRS detection
- **Main thread skip**: `load-url.ts` and `load-file.ts` skip all map rendering (no source, no layers, no bounds fitting, no hover/click handlers) for non-spatial results; datasets are tracked and the panel refreshes
- **Table Only advanced option**: new checkbox in the shared advanced options panel (`ui/advanced-options.ts`) that forces table-only loading regardless of whether geometry is present; disables CRS, renderer, and column override fields when checked; wired through both DataControl and UploadControl
- **Dataset panel differentiation**: non-spatial datasets display with a gray left border, show "N rows" instead of "N features", and hide the map-pin/layers button; table preview works unchanged since `getPreviewRows` has no geometry filter

### Join operation

- **`join` operation type**: new binary operation that merges properties from a source (tabular/lookup) dataset into a target (spatial) dataset by matching a shared key field; `inputs[0]` is the source, `inputs[1]` is the target; output geometry comes from the target dataset; config params: `sourceKey`, `targetKey`, and optional `mode`
- **Join modes**: `left` (default) keeps all target features and merges source properties where keys match; `inner` keeps only target features that have a matching source row
- **SQL implementation**: `json_extract_string` key matching with `json_merge_patch` for property merging; runs as a single INSERT..SELECT with LEFT JOIN or INNER JOIN depending on mode; parameterized dataset IDs with validated config-sourced JSON path expressions for key fields
- **Validation**: `validateJoinParams` in `validators/operations.ts` requires non-empty string `sourceKey` and `targetKey`; validates `mode` as `inner` or `left` when present; operation-level validation requires `params` block
- **Type additions**: `JoinParams` interface in `types.ts`; `join` added to `BinaryOperationType` union; `BINARY_OPERATIONS` array in `parser.ts` updated; worker dispatch case in `worker.ts` routes to `computeJoin`

### Operation Builder join support

- **Join inputs in Operation Builder**: selecting `join` as the operation type renders "Source (tabular)" and "Target (spatial)" dataset dropdowns instead of the generic "Input A" / "Input B" labels; dataset selection triggers async column loading for the key selectors
- **Key field selectors**: `sourceKey` and `targetKey` rendered as `<select>` dropdowns populated from dataset column metadata via `getPropertyKeys` worker RPC; loading state shown while columns are fetched; error and empty states handled
- **Mode selector**: dropdown with "Left (keep all target features)" and "Inner (matches only)" options; defaults to left; `mode` param omitted from generated YAML when set to `left` (the default)
- **Validation**: `validateInputs` requires both `sourceKey` and `targetKey` selections before allowing execution; `buildParams` assembles the join params object for YAML preview and execution

### Clear session performance

- **DROP instead of DELETE in clearOPFS**: `workerClearOPFS` in `worker.ts` now uses `DROP TABLE IF EXISTS` (O(1) metadata operation) instead of `DELETE FROM` (O(N) row scan) when clearing session data; large datasets that previously hung for minutes now clear in under a second; tables are recreated by `initSchema` on page reload
- **DROP instead of DELETE in deleteAllDatasets**: `deleteAllDatasets` in `datasets.ts` uses the same DROP + `initSchema` recreate pattern instead of bulk DELETE, avoiding the O(N) row scan and post-delete VACUUM
- **Deferred checkpoint/vacuum for batch deletion**: `deleteDataset` gains a `skipMaintenance` flag; `teardownAll` passes `skipMaintenance=true` when selectively deleting non-file datasets in a loop, then runs a single checkpoint + vacuum after all deletions complete instead of per-dataset

### StorageControl icon and shortcut

- **StorageControl icon**: replaced `databaseIcon` with new `hardDrivesIcon` (Phosphor HardDrives Regular) to free the database icon for the upcoming DatasetControl; new icon file at `icons/hard-drives.ts` exported from the icon index
- **Keyboard shortcut**: changed from T to G (storaGe); button title updated to "Storage (G)"; shortcut binding in `map.ts` updated

## v0.9.0 - 2026-04-06

### Config Schema: `renderer` field

- **`renderer` field on LayerConfig**: optional `renderer: maplibre | deckgl` property on layer config entries (defaults to `maplibre`); new `RendererType` union type exported from `types.ts`; `VALID_RENDERERS` constant exported from `parser.ts` for runtime validation
- **`deckProps` passthrough on LayerConfig**: optional `deckProps` object for raw deck.gl layer properties (color accessors, radius scale, etc.) spread directly into the deck.gl layer spec without content validation; only the container type is checked (must be a non-null, non-array object)
- **Validation rules**: invalid `renderer` values produce a parse error listing valid options; `deckProps` present without `renderer: 'deckgl'` produces a console warning (non-blocking); `GeneratedLayerConfig` in the config generator updated to preserve both fields through YAML round-trips
- **Layer validator unit tests**: new `layers.test.ts` covering all existing layer validation rules (required fields, optional field constraints, duplicate IDs, source references) and the new renderer/deckProps rules (valid values, invalid values, type errors, warning on mismatched renderer)

### `MapboxOverlay` manager

- **Singleton DeckGLManager**: module-level singleton (`deckgl/manager.ts`) holding a single `MapboxOverlay` instance; composites all deck.gl layers into one WebGL context rendered on top of MapLibre; exposes `addLayer`, `removeLayer`, `updateLayer`, `setLayerVisibility`, `destroy`, `isInitialized`, `hasLayer`
- **Lazy loading**: deck.gl packages (`@deck.gl/core`, `@deck.gl/layers`, `@deck.gl/mapbox`) loaded via dynamic `import()` only when the first deck.gl layer is requested; no deck.gl code in the initial bundle; concurrent initialization guarded by shared promise
- **Teardown integration**: `teardownAll()` dynamically imports the manager and calls `destroy()` only if deck.gl was initialized; overlay removed from map, registry cleared, module state reset
- **Manual test**: temporary hardcoded `GeoJsonLayer` block in `map.ts` (gated by `DECKGL_TEST` constant) renders the first loaded dataset via the deck.gl overlay to verify compositing onto MapLibre (removed in Layer creation branch)

### Layer creation branch and registry

- **Renderer branch in `executeLayersFromConfig`**: branches on `layerConfig.renderer` -- MapLibre path unchanged, deck.gl path fetches GeoJSON from DuckDB via `getFeaturesAsGeoJSON` and constructs a `GeoJsonLayer` spec; `buildDeckGLPropsFromConfig` translates `paint` properties (fill-color, line-color, line-width, circle-radius, opacity) to deck.gl accessors; `deckProps` spread on top for user overrides; function signature changed from sync to async
- **Renderer registry** (`deckgl/registry.ts`): `Map<string, RegistryEntry>` keyed by layer ID, storing `{ renderer, source }` tuples; `registerLayer` called during layer creation, `unregisterLayer` during deletion, `clearRegistry` during teardown; `getRenderer` returns `'maplibre'` for unregistered IDs (safe default); `getLayersByDataset` filters by source and optional renderer type
- **Color conversion utilities** (`deckgl/color.ts`): `hexToRGBA` (supports #RGB, #RRGGBB, #RRGGBBAA with optional alpha override), `rgbaToHex`, `buildDeckColorProps` for translating CSS hex colors to deck.gl RGBA arrays

### Popup/hover parity

- **deck.gl hover tooltips**: `buildDeckHoverCallback` in `popup.ts` returns a closure wired as the deck.gl `onHover` callback; drives the existing shared hover tooltip singleton with layer name and optional `tooltip` field values; cursor set to pointer on hover, reset on leave
- **deck.gl click popups**: `buildDeckClickCallback` in `popup.ts` returns a closure wired as the deck.gl `onClick` callback; drives the existing shared click popup singleton with full feature property table; dismisses hover tooltip on click
- **Shared popup behavior**: both renderers write to the same `maplibregl.Popup` instances; deck.gl layers register in `hoverRegistry` and `clickRegistry` so config generation, `getTooltipFields`, `getHoverOptions`, `updateHoverLabel`, and `removeFeatureHandlers` all work identically
- **Auto-pickable**: `recomposite()` in the deck.gl manager auto-injects `pickable: true` when `onHover` or `onClick` exists in layer props; explicit `pickable` in `deckProps` overrides
- **Cleanup fix**: `deleteDatasetWithLayers` now calls `removeFeatureHandlers` for deck.gl layers before removal, preventing stale registry entries

### Layer control renderer-awareness

- **`visibility.ts`**: dual dispatch -- MapLibre layers via `setLayoutProperty`, deck.gl layers via dynamic import of manager `setLayerVisibility`; registry is authoritative for deck.gl layers since they don't appear in MapLibre's style
- **`color.ts`**: `updateLayerColor` dispatches to deck.gl manager `updateLayer` with `buildDeckColorProps` for deck.gl layers; `isColorPickerEnabled` returns true for deck.gl layers (no expression support, always editable)
- **`delete.ts`**: `deleteDatasetWithLayers` removes deck.gl layers via manager `removeLayer` and unregisters all layers from the registry
- **`style.ts`**: `getEditableProperties` returns all-true for deck.gl datasets; `applyStyleToMap` dispatches width and radius changes to deck.gl manager; `mapStyleToDeckProps` translates slider properties to deck.gl layer props
- **`map.ts`**: deck.gl layers skip MapLibre popup/hover attachment (wired in `addDeckGLLayerFromConfig` instead); `sourceNameMap` hoisted before `executeLayersFromConfig` and passed through for deck.gl label resolution; config color/style sync blocks skip deck.gl-rendered datasets; temporary `DECKGL_TEST` block removed
- **`teardown.ts`**: `clearRegistry` called alongside `destroy` during teardown
- **Test page**: `/test-deckgl` dev route with `test-deckgl-config.yaml` -- parks via MapLibre, streets via deck.gl on the same map; verifies renderer coexistence and panel actions
- **Unit tests**: 20 new tests -- registry operations (register, unregister, getRenderer, isDeckGL, getLayersByDataset, clearRegistry) and color conversions (hexToRGBA, rgbaToHex, buildDeckColorProps)

### Arrow binary data path

- **`getFeaturesAsBinaryCollection`**: new function in `features.ts` that queries DuckDB with `ST_AsWKB(geometry)` and flattened property columns, parses WKB via a lightweight coordinate extractor (`db/wkb.ts`), and builds a `BinaryFeatureCollection` with flat typed arrays (positions, featureIds, pathIndices, polygonIndices, numericProps) that deck.gl `GeoJsonLayer` consumes directly in binary mode
- **WKB parser** (`db/wkb.ts`): standalone coordinate extractor for Point, LineString, Polygon, MultiPoint, MultiLineString, MultiPolygon; flattens Multi* types into singular geometry entries; reads byte order, geometry type, and coordinate sequences from standard WKB binary
- **Worker RPC**: `GetFeaturesAsBinaryRequest` message type; worker handler returns the binary collection via `respondTransfer` with all typed array buffers as Transferables for zero-copy transfer to the main thread
- **deck.gl integration**: `addDeckGLLayerFromConfig` in `layers.ts` calls `getFeaturesAsBinary` instead of `getFeaturesAsGeoJSON`; the `BinaryFeatureCollection` passes directly as `data` to `GeoJsonLayer`, which auto-detects binary format; GeoJSON fallback on error preserves backwards compatibility
- **Hover/click handler fix**: MapLibre `mousemove` and `click` handlers in `popup.ts` now filter deck.gl layer IDs from `queryRenderedFeatures` calls via the renderer registry, preventing errors when both renderers coexist
- **Test**: A/B comparison at `/test-deckgl` -- Vancouver Water Distribution Mains (66,000 line features via GeoParquet) rendered simultaneously as MapLibre/GeoJSON (orange) and deck.gl/binary (blue); deck.gl layer displays instantly on toggle while MapLibre renders progressively

### Binary data picking fix

- **`info.object` resolution for binary data**: deck.gl v9 `GeoJsonLayer` does not populate `info.object` during picking when data is a `BinaryFeatureCollection`; hover/click callbacks now resolve properties via a `globalProperties` lookup array (indexed by `globalFeatureId`) built from the binary data at layer creation time; GeoJSON fallback path unchanged
- **`buildGlobalProperties` utility**: exported from `popup.ts`; iterates `globalFeatureIds` and `featureIds` across all geometry types to build a flat property lookup from the binary data already on the main thread; zero memory duplication (stores references to existing property objects)
- **Popup z-index fix**: deck.gl `MapboxOverlay` in non-interleaved mode renders an absolutely-positioned canvas that covered popup DOM elements; added `z-index: 3` to `.maplibregl-popup` so both hover tooltips and click popups render above the deck.gl overlay

### Renderer selection in Load Data / Upload File GUI

- **Renderer dropdown in advanced options**: new Renderer `<select>` in the shared advanced options panel (`ui/advanced-options.ts`) with three choices -- Auto (default), MapLibre, deck.gl; Auto resolves to MapLibre for now (auto-promote logic planned for v0.9.3); `AdvancedOptionsValues` gains an optional `renderer` field typed as `RendererType`
- **`addDatasetToMapDeckGL` in `shared.ts`**: new async function for the deck.gl GUI load path; fetches data from DuckDB via binary path with GeoJSON fallback, builds color/style props via `buildDeckColorProps`, wires hover/click callbacks, delegates to the deck.gl manager, and registers in the renderer registry; all deck.gl imports are dynamic to preserve lazy-loading
- **Renderer branching in loaders**: `load-file.ts` and `load-url.ts` resolve `options.renderer` (deck.gl explicit, everything else to MapLibre) and branch -- deck.gl path calls `addDatasetToMapDeckGL` and skips MapLibre popup attachment (deck.gl wires its own callbacks); MapLibre path unchanged
- **Controls passthrough**: `DataControl` and `UploadControl` pass `renderer` from `advancedOptions.getValues()` through to the load functions; `LoadDataOptions` interface gains `renderer?: RendererType`

### deck.gl example

- **Meteorite Landings example**: new example showcasing deck.gl rendering with 38,000+ NASA meteorite landing records loaded from GeoParquet; amber points on dark basemap with binary Arrow data path for instant rendering; new "deck.gl" group in the examples sidebar

### Config editor horizontal scrolling

- **Body overflow**: `.config-viewer-body` changed from `overflow-y: auto` to `overflow: auto`; view mode pre content with `white-space: pre` now scrolls horizontally without any JS changes
- **Edit mode auto-width**: `autoGrow` extended to measure `textarea.scrollWidth` and set the editor wrapper width to `max(contentWidth, bodyClientWidth)`; the body div scrolls both axes while the two-layer highlight/textarea architecture stays unchanged
- **Scroll position preservation**: `scrollLeft` saved and restored alongside `scrollTop` across view/edit transitions and async re-highlight passes
- **Resize recalculation**: `autoGrow` called on resize `pointerup` when editing so editor width adjusts after panel resize
